# Active Source Material

This folder contains the raw inputs that existed in `active-plans/` before the reorganization:

- draft plans
- copied vendor or ecosystem documentation
- research dumps
- failure notes from downstream repos
- exploratory validation/tooling notes

These files remain active documentation inside `active-plans/`, but they are not the canonical implementation plans anymore.

Use the numbered plan folders for active work. Use `_source-material/` when you need to trace the origin of a planning decision, preserve copied research in near-original form, or extract more detail during implementation.

Rules:

1. Do not treat `_source-material/` as a future-removal queue.
2. Do not hide canonical implementation instructions here; promote those into the relevant numbered folder `README.md` or `references/` note.
3. If a source note becomes fully superseded, add a pointer to the replacement plan or report so provenance stays clear.
4. If a source note turns into active plan-local guidance, distill it into that plan's `references/` folder and keep the original here only when the provenance still matters.
